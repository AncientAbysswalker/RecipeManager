Introduction Last updated: 2022-02-17

As part of my Raviolé recipe manager project I needed a method to store and retrieve recipes data. Over the course of the project the database and associated endpoints expanded and this documentation serves to outline its usage. The database has two fairly specific data types that are used, and these are outlined below.

All endpoints are extensions of the base URL http://www.raviole.cerberus-heuristics.com

Recipe Object

The Recipe Object is responsible for storing all of the information associated with the recipe. In the case of images, the image names (uploaded hash) are stored for referencing the CDN. An example of a Recipe Object is shown below. Note that the owner and _id properties are reserved, and trying to modify them will be ignored.

{
	"_id": "60bc2199d45b8e03541ff799",
	"owner": "213bc9b8f7545e0609d991f4",
	"name": "Fried Eggs",
	"time_active": 5,
	"time_total": 5,
	"ingredients": [
		{
			"name": "Egg", 
			"quantity": "2", 
			"unit": "Each"
		},
		{
			"name": "Salt", 
			"quantity": "1/2", 
			"unit": "tsp"
		}
	],
	"instructions": [
		{
			"type": 0,
			"title": "Preparation",
			"contents": [
				{
					"type": 1,
					"text": "Place eggs and salt into bowl, mixing until yolk and salt are fully integrated with whites.",
				}
			]
		},	
		{
			"type": 0,
			"title": "Cooking",
			"contents": [
				{
					"type": 1,
					"text": "Heat non-stick pan on medium heat. Pour egg mixture into pan, stirring continuously until cooked.",
				}
			]
		}	
	],
	"images": ["fc8cf377-6056-476f-9597-6fef05f3c9b5.jpg"],
	"tags": ["breakfast","eggs"],
	"yield": "1 meal"
}

Collections Object

The Collections Object is fairly straight-forward. The collections object consists of a key for every Collection name, with the value being a list of all recipe ids that the collection contains. An example is shown below.

{
	"Collection 1": [
		"60bc2199d45b8e03541ff799", 
		"213bc9b8f7545e0609d991f4"
	],
	"Collection 2": [
		"997ff14530e8b54d9912cb06", 
		"4f199d9060e5457f8b9cb312"
	]
}

User Access Control Endpoint

Several other API calls require for an active session to exist between the server and the user's local machine. Eash session is associated with a username that exists in the system. The following define the behavior for login, logout, signup, and checking session status.

GET/sessionGet the current session status

Response

data.loggedInbool

data.usernamestring

data.userIdstring<12>

Current session status. If a session is active, the logged in user's username and id will also be returned.

POST/signupCreate a new user and log into a session with them

Request Body

body.usernamestring

body.passwordstring

body.emailstring

Username, password, and email to use to create a new user. Username and email will be checked to ensure they do not already exist.

Response

data.loggedInbool

data.usernamestring

data.userIdstring<12>

Current session status including the logged in user's username and id.

Note: The session is automatically created as part of this process.

POST/loginLog into a session using a username and password

Request Body

body.usernamestring

body.passwordstring

Username and password to use to log in and create a new session.

Response

data.loggedInbool

data.usernamestring

data.userIdstring<12>

Current session status including the logged in user's username and id.

Note: The session is automatically created as part of this process.

POST/logoutLog out of a sessionActive Session Required

Session Specific Info

A valid session must exist and be passed as part of the request headers. If a session exists in the headers, that session will be ended.

Recipes Endpoint

Responsible for all most recipe-related behavior. Recipes can be retrieved fully or in part, can be added, can be updated, and can be deleted. Most of these actions require an active session to execute for security.

GET/recipes{?optional_params}Return a JSON representation of all recipes in the database

Query Parameters

fieldsstring

example:tags, time_active, images, headers

example usage:/recipes/60bc2199d45b8e03541ff799?fields=name&fields=tags


Any specific fields that should be returned by the query. Default if excluded is all fields. All properties of the RecipeObject are valid strings. The alias "headers" is also permitted, which indicates that the name, images, and tags should be returned.

Response

dataRecipeObject[]

List of recipe objects, with any filtering from query parameters.

GET/recipes/{recipe_id}{?query_params}Return a JSON representation of a single recipe in the database by its id

Path Parameters

recipe_idstring <12 bytes>

example:60bc2199d45b8e03541ff799

example usage:/recipes/60bc2199d45b8e03541ff799


The database id of the recipe to be queried.

Query Parameters

fieldsstring

example:tags, time_active, images, headers

example usage:/recipes/60bc2199d45b8e03541ff799?fields=name&fields=tags


Any specific fields that should be returned by the query. Default if excluded is all fields. All properties of the RecipeObject are valid strings. The alias "headers" is also permitted, which indicates that the name, images, and tags should be returned.

Response

dataRecipeObject

Recipe object requested by its id, with any filtering from query parameters.

POST/recipesCreate a new recipe in the databaseActive Session Required

Request Body

bodyRecipeObject

Recipe object to be added to the database. Properties of the recipe object should be properties of the request's body.

Response

dataRecipeObject

The recipe object added to the database, which includes the id of the new recipe in the database.

Session Specific Info

A valid session must exist and be passed as part of the request headers or the request to add the recipe to the database will be rejected.

Note: As this action is linked to the active session and its user, the new recipe id will automatically be added to the user's saved recipes.

PUT/recipes/{recipe_id}Update a recipe in the database by its idActive Session Required

Path Parameters

recipe_idstring <12 bytes>

example:60bc2199d45b8e03541ff799

example usage:/recipes/60bc2199d45b8e03541ff799


The database id of the recipe to be queried.

Request Body

bodyRecipeObject

Recipe object to be added to the database. Properties of the recipe object should be properties of the request's body.

Response

dataRecipeObject

The recipe object that was modified in the database.

Session Specific Info

A valid session must exist and be passed as part of the request headers or the request to modify the recipe in the database will be rejected.

DELETE/recipes/{recipe_id}Delete a recipe from the database by its idActive Session Required

Path Parameters

recipe_idstring <12 bytes>

example:60bc2199d45b8e03541ff799

example usage:/recipes/60bc2199d45b8e03541ff799


The database id of the recipe to be queried.

Response

dataRecipeObject

The recipe object that was deleted from the database.

Session Specific Info

A valid session must exist and be passed as part of the request headers or the request to modify the recipe in the database will be rejected.

Images Endpoint

The images endpoint acts as a Content Delivery Network - enabling images to be uploaded and retrieved.

GET/images/{image}Return an image from the database

Query Parameters

imagestring

example:fc8cf377-6056-476f-9597-6fef05f3c9b5.jpg

example usage:/images/fc8cf377-6056-476f-9597-6fef05f3c9b5.jpg


The database location of the image to be returned.

POST/images/uploadUpload new images to the database

Form Data

The images to be uploaded must be added to a list, which must be added to a FormData object under property images. This FormData object is passed to the query to upload the images.

Collections Endpoint

When recipes are created they are created with an owner. In addition to owning recipes, a user can save recipes created by others. Thus there is a subset of all recipes that are linked to that user through either ownership or saving. Collections extend this to allow the user to make named groups of recipes for quick reference, like "Favorite Breakfasts".

GET/user-saved-recipes{?optional_params}Return a JSON representation of all recipes owned by the userActive Session Required

Query Parameters

fieldsstring

example:tags, time_active, images, headers

example usage:/user-saved-recipes?fields=name&fields=tags


Any specific fields that should be returned by the query. Default if excluded is all fields. All properties of the RecipeObject are valid strings. The alias "headers" is also permitted, which indicates that the name, images, and tags should be returned.

Response

dataRecipeObject[]

List of recipe objects, with any filtering from query parameters.

Session Specific Info

A valid session must exist and be passed as part of the request headers or no data will be returned.

GET/userdata/saved-recipes/{recipe_id}Return whether a recipe is saved by the userActive Session Required

Path Parameters

recipe_idstring <12 bytes>

example:60bc2199d45b8e03541ff799

example usage:/userdata/saved-recipes/60bc2199d45b8e03541ff799


The database id of the recipe to be queried.

Response

databool

Whether or not the user linked to the session has saved this recipe.

Session Specific Info

A valid session must exist and be passed as part of the request headers or false will always be returned.

PUT/userdata/saved-recipes/{recipe_id}Set whether a recipe is saved by the userActive Session Required

Path Parameters

recipe_idstring <12 bytes>

example:60bc2199d45b8e03541ff799

example usage:/userdata/saved-recipes/60bc2199d45b8e03541ff799


The database id of the recipe to be queried.

Request Body

bodybool

Whether the recipe identified by the provided id should be considered as saved for the user associated with the active session.

Session Specific Info

A valid session must exist and be passed as part of the request headers or the request will be rejected.

GET/userdata/collectionsReturn the collections of the userActive Session Required

Response

data.userCollectionsCollections

Mapping of collections that the user has defined.

Session Specific Info

A valid session must exist and be passed as part of the request headers or an empty JSON object will always be returned.

PUT/userdata/collectionsOverwrite the collections for the userActive Session Required

Request Body

bodyCollections

Mapping of collections for the user. Will overwrite the previous state. Handling for adding, modifying, and deleting collections is handled client-side.

Session Specific Info

A valid session must exist and be passed as part of the request headers or the request will be rejected.
